# Teoría de Herencia en Programación (Python)

La **herencia** en programación se basa en el mismo principio que la herencia en la vida real. Así como heredamos características físicas o rasgos de nuestros padres y abuelos (como el color de ojos, la voz o la estatura), en programación una clase puede heredar **características y comportamientos** de otra.

## Concepto básico

En programación orientada a objetos, una clase puede heredar atributos y métodos de otra clase. La clase que hereda se llama **clase hija (o subclase)**, y la clase de la que se hereda se llama **clase padre (o superclase)**.

La herencia permite reutilizar código y establecer una jerarquía entre clases.

## Ejemplo conceptual

Imaginemos una clase `Persona` y una clase `Empleado`:

```python
class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

class Empleado(Persona):
    def __init__(self, nombre, edad, sueldo):
        super().__init__(nombre, edad)
        self.sueldo = sueldo
```

En este ejemplo:

- `Persona` es la **clase padre**.
- `Empleado` es la **clase hija** que **extiende** a `Persona`.
- `Empleado` hereda los atributos `nombre` y `edad`, y además añade el atributo `sueldo`.

## Herencia implícita de `object`

En Python, **todas las clases heredan indirectamente de la clase `object`**, incluso si no lo especificamos.  
Por lo tanto, la jerarquía sería:

```
object → Persona → Empleado
```

Esto significa que `Empleado` hereda las características de `Persona`, y a su vez `Persona` hereda de `object`.

## Herencia múltiple

Python permite la **herencia múltiple**, es decir, una clase puede heredar de más de una clase padre.  
Por ejemplo:

```python
class A:
    pass

class B:
    pass

class C(A, B):
    pass
```

Aquí, la clase `C` hereda de `A` y de `B` simultáneamente.

## Ventajas de la herencia

- **Reutilización de código**: se pueden aprovechar las características de clases ya existentes.
- **Mantenimiento más sencillo**: si una modificación se realiza en la clase padre, las clases hijas la heredan automáticamente.
- **Organización jerárquica**: permite estructurar el código de forma lógica y escalable.

## Ejemplo práctico

Siguiendo el ejemplo de `Persona` y `Empleado`:

```python
persona = Persona("Ana", 30)
empleado = Empleado("Luis", 28, 50000)

print(empleado.nombre)  # Heredado de Persona
print(empleado.sueldo)  # Propio de Empleado
```

**Salida:**

```
Luis
50000
```

El objeto `empleado` accede tanto a los atributos heredados (`nombre`, `edad`) como a los propios (`sueldo`).

La flecha indica que `Empleado` **hereda** de `Persona`.

---

**Conclusión:**  
La herencia es uno de los pilares fundamentales de la programación orientada a objetos. Permite construir sistemas más organizados, escalables y fáciles de mantener, fomentando la reutilización del código y la extensión natural de funcionalidades.
